Preguntas a responder

¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor?

Para asegurarse que los datos que se ingresaron son correctos en cuanto al contenido que sea conforme al formate que se pide, que cumpla con los requisitos y tambien evitar que ingresen datos no validos que hagan que el servidor tenga una falla

¿Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript?

Se puede saltar la validacion entrando al codigo que se puede acceder desde la inspeccion de codigo y ahi eliminar las partes de codigo que realizan la validacion. Otra forma es desactivar el JS desde la configuracion del navegador.

Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica?

Porque las validaciones si se deben de realizar pero no del lado del cliente donde tiene acceso al codigo, sino que estas validaciones se deberian de realizar del lado del servidor y mandar feedback al usuario
